## What is this?

Checkboxes are tools that customize configurations. This control has three states: unchecked, checked and indeterminate. They reflect the selection of an option and its execution usually requires another control, or confirmation.

<PageSection
  id={tabToc.overview[11].id}
  title={tabToc.overview[11].value}
  explanation="To implement a checkbox that controls the state of a set of checkboxes you will need to combine the checkbox's three states: checked, unchecked, and indeterminate."
>

```jsx live
function Example() {
  function useTreeState({ values }) {
    const { value: group, setValue: setGroup } = useCheckboxState({
      initialValue: [],
    })
    const { value: items, setValue: setItems } = useCheckboxState({
      initialValue: [],
    })

    // updates items when group is toggled
    React.useEffect(() => {
      if (group === true) {
        setItems(values)
      } else if (group === false) {
        setItems([])
      }
    }, [group, setItems, values])

    // updates group when items is toggled
    React.useEffect(() => {
      if (items instanceof Array && items.length === values.length) {
        setGroup(true)
      } else if (items instanceof Array && items.length) {
        setGroup('indeterminate')
      } else {
        setGroup(false)
      }
    }, [items, setGroup, values])

    return { group, items, setItems, setGroup }
  }

  const values = React.useMemo(() => ['Apple', 'Orange', 'Watermelon'], [])
  const { group, setGroup, items, setItems } = useTreeState({ values })

  return (
    <CheckboxGroup label="Fruits" id="fruits-checkbox-group" direction="column">
      <Checkbox
        label="Select All (Group Control)"
        state={{ value: group, setValue: setGroup }}
      />
      {values.map((fruit, key) => (
        <Checkbox
          state={{ value: items, setValue: setItems }}
          value={fruit}
          label={fruit}
        />
      ))}
    </CheckboxGroup>
  )
}
```

<Section direction="row">

## Best Practices

<OrderedList>
  <OrderedListItem>
    Multiple choices.
  </OrderedListItem>
  <OrderedListItem>
    When multiple sub-options are grouped under a parent option and each sub-option is in a different state, the parent option should be in the indeterminate state.
  </OrderedListItem>
  <OrderedListItem>
    Clear visual state for selected and unselected options.
  </OrderedListItem>
  <OrderedListItem>
    Use a checkbox for a list of related items.
  </OrderedListItem>
  <OrderedListItem>
    Use a checkbox when binary options are available, such as yes or no choices.
  </OrderedListItem>
  <OrderedListItem>
    Use titles to group related controls.
  </OrderedListItem>
  <OrderedListItem>
    Choose words that are aligned with our ecommerce vocabulary.
  </OrderedListItem>
  <OrderedListItem>
    Write the current state of the control, to improve accessibility.
  </OrderedListItem>
  <OrderedListItem>
    Add descriptions if the control is critical, or complex.
  </OrderedListItem>
  <OrderedListItem>
    Lists of controls should be vertical for better scannability.
  </OrderedListItem>
  <OrderedListItem>
    Checkbox items should be unchecked by default.
  </OrderedListItem>
  <OrderedListItem>
    Use a disabled checkbox only if in some condition it becomes enabled. Avoid multiple disabled inputs in the same screen.
  </OrderedListItem>
  <OrderedListItem>
    Look for labels that are already used in other pages in the Admin and avoid using new terms when the semantic is very similar.
  </OrderedListItem>
  <OrderedListItem>
    After a checkbox, include a confirmation button to require an additional action that will confirm the selection.
  </OrderedListItem>
</OrderedList>
