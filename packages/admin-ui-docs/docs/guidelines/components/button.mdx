- `Button` has the role `button`.
- When `Button` is in a focused state, pressing <kbd>Space</kbd> and <kbd>Enter</kbd> trigger its action.
- If `disabled` prop is `true`, `Button` has `disabled` and `aria-disabled` attributes set to `true`.
- When using an Icon-only button, you must set the `aria-label` property to make it accessible.

```jsx
function Example() {
  return (
    <Button
      icon={<IconHeart title="Favorite" />}
      aria-label="Favorite button"
    />
  )
}
```

<Panel id="designReference">

## Do's

1. Buttons should be completely straight-forward. Absolutely no room for ambiguity.
2. The behavior they trigger must reflect the exact action that is written.
3. When using icons as buttons, be mindful of accessibility, always include descriptions in the alt tag.
4. Pay attention to the icon's use in other screens, be consistent of what action it represents.

## Types

### Primary

1. This option should be used for the single most important action of the page.
2. The icon must reinforce the importance of the task.
3. Use this on page headers and confirmation modals.
4. Use it only once per page / modal.

### Secondary

1. This option should be used for non-urgent, or optional actions.
2. Use this option for page headers, lists and forms.

### Tertiary

1. This option should be used for non-urgent, or optional actions.
2. Use this option for page headers, tables, lists and forms.
3. Can be used in tight spaces such as modals, sidebars or cards.
